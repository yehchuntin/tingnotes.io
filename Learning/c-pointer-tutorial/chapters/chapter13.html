Dynamic Memory Allocation

動態記憶體配置的完整指南

🔧 malloc (Memory Allocation)

malloc 是 C 語言中最基本的動態記憶體配置函式，用於在堆積區（Heap）配置指定大小的記憶體空間。

🔧 函式用法

void* malloc(size_t size);

📌 參數說明

  • size → 總共要配置的記憶體大小（位元組）

🎯 功能

  • 配置一塊記憶體空間
  • 內容不初始化（為垃圾值）
  • 回傳 void* 指標，需轉型 ex: (int *)、(char *)
💡 範例 
int *p = (int *)malloc(3 * sizeof(int));

♻️ 回收空間

free(p);
⚠️ 重要提醒
  • 使用 malloc 配置的記憶體必須使用 free 釋放
  • 配置失敗時會回傳 NULL，使用前應檢查
  • 記憶體內容為隨機值，不是 0

🆕 calloc (Contiguous Allocation)

calloc 是另一個動態記憶體配置函式，與 malloc 的主要差異在於它會自動將配置的記憶體初始化為 0。

🔧 函式用法

void* calloc(size_t num, size_t size);

📌 參數說明

  • num → 要配置的元素個數 (number of elements)
  • size → 每個元素的大小（使用 sizeof(type)）

🎯 功能

  • 配置一塊連續記憶體空間
  • 內容自動初始化為 0
  • 回傳 void* 指標，需轉型 ex: (int *)、(char *)
💡 範例 
int *p = (int *)calloc(3, sizeof(int));

🔍 calloc vs malloc

  • calloc：記憶體自動初始化為 0，適合需要乾淨記憶體的情況
  • malloc：記憶體內容為隨機值，效能較佳

📏 realloc (Reallocate Memory)

realloc 用於調整已配置記憶體區塊的大小，可以增加或減少記憶體空間，是動態陣列實作的關鍵函式。

🔧 函式用法

void* realloc(void* ptr, size_t new_size);

📌 參數說明

  • ptr → 原本的記憶體指標（可為 NULL）
  • new_size → 新的記憶體大小（位元組）

🎯 功能

  • 調整已配置記憶體區塊的大小：
    • 增加：可能搬移位置（內容會保留）
    • 減少：前段保留，後段釋放
  • ptr == NULL → 等同 malloc
  • new_size == 0 → 等同 free
💡 範例 
int *p = (int *)malloc(3 * sizeof(int));
p = (int *)realloc(p, 5 * sizeof(int));
⚠️ realloc 使用注意事項
  • 可能會改變記憶體位址，原指標可能失效
  • 失敗時回傳 NULL，但原記憶體不會被釋放
  • 建議先用臨時變數接收結果再賦值

📊 函式比較總結

以下表格總結了三個動態記憶體配置函式的主要特性與差異。

函式 自動初始化 可調整大小 備註
malloc ❌ 否 ❌ 否 需自行初始化
calloc ✅ 是 ❌ 否 初始化為 0
realloc ❌ 否 ✅ 是 調整大小並保留原資料

🎯 使用場景建議

  • malloc：一般用途，效能優先
  • calloc：需要乾淨記憶體（如陣列）
  • realloc：動態調整大小（如動態陣列）
💡 記憶體管理最佳實務
  • 每次 malloc/calloc 都要對應一次 free
  • 使用後將指標設為 NULL 避免誤用
  • 檢查回傳值是否為 NULL（配置失敗）
  • 避免重複釋放同一塊記憶體
