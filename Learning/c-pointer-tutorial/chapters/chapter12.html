C 函式執行與記憶體配置

在 C 語言中，記憶體主要分為四個重要的區域，每個區域都有其特定的用途和特性。

[圖片位置：C函式執行與記憶體配置圖]
📂 記憶體分區（高→低位址）
  • Heap：動態配置 malloc
  • Stack：函式的參數與區域變數
  • Static/Global：全域變數、static變數
  • Code：可執行程式碼

💡 變數分布位置

變數 所屬區域 位於記憶體哪一區
total 全域變數 Static / Global 區
a, b main 的區域變數 Stack
x, y, z SquareOfSum 的區域變數 Stack
x Square 的參數 Stack（呼叫 Square 時）

📌 執行流程

  • main() 呼叫 SquareOfSum(a, b) → 建立新的 Stack frame
  • SquareOfSum() 呼叫 Square(x + y) → 再建立一個 Stack frame
  • 函式結束 → 一層一層釋放 Stack
🧠 總結一句話記住
  • total 是資料（變數）→ 放 Global 區
  • Square() 和 SquareOfSum() 是程式碼 → 放 Text 區
  • 它們和main() 的順序無關，因為「寫在上面 ≠ 放在上面」
⚠️

Stack Overflow 是什麼？

在 C 語言中，Stack Overflow（堆疊溢位）是指：

程式使用過多的 stack 記憶體，導致超出系統所配置的限制，進而程式崩潰或產生未定義行為。

🔧 Stack 是什麼？

Stack 是 C 程式中用來儲存：

  • 函式的的參數
  • 區域變數（local variables）
  • 回傳位址（return address）

它的記憶體大小通常有限（通常幾 MB 而已）

⚠️ 什麼情況會發生 Stack Overflow？
情況 範例 說明
遞迴呼叫太深 func() { func(); } 每次呼叫都會用一個新的 stack frame
區域變數太大太 int A[10000000]; 區域陣列太大，超過 stack 容量
多層函式太深呼叫 太多函式疊起來，每層都有很多區域變數 累積太多 stack frame 導致沒位

🔧 如何避免 Stack Overflow？

  • 避免無限遞迴
  • 不要在 Stack 宣告太大陣列（改用 malloc 動態配置 → 放在 Heap）
  • 避免過深呼叫（即 DFS）

🔄 Stack Overflow vs Heap Overflow？

分類 Stack Overflow Heap Overflow
發生位置 函式呼叫過度 動態配置過度（malloc）
常見原因 遞迴太深、陣列太大 寫入超出配置的記憶體範圍
解決方式 減少遞迴/改 malloc 加強記憶體邊界檢查

Heap 是什麼？該如何使用它？

在 C++程式中，Heap（堆積區）是一塊專門給程式動態配置記憶體的空間，也就是我們「需要多少、什麼時候配置、何時釋放」都由程式設計師來決定。

[圖片位置：Heap記憶體配置圖]

🔧 Heap 的基本概念

  • Heap 是應用程式的一部分記憶體空間，由作業系統分配，容量大。
  • Heap 裡的資料不像 Stack 會自動釋放，必須手動處理。
  • 常用於儲存：陣列、結構、鏈結串列等大型或不定大小的資料。

在上圖中，Heap 是記憶體的藍色區塊，位於 Stack 之上，是整體 Application's memory 的一部分。

🛠️ C/C++ 中的 Heap 使用方式

語言 使用方式 範例
C 使用函式 malloc、calloc、realloc、free 來操作記憶體 int *p = malloc(4 * sizeof(int));
C++ 使用運算子 new、delete 來操作記憶體 int *p = new int[4];

這些函式與運算子都會直接在 Heap 區域調配記憶體。

📌 注意事項

  • Heap 使用後必須釋放：free()（C）、delete（C++）
  • 忘記釋放會造成記憶體洩漏（Memory Leak）。(後面會提到!)）
  • Stack 是系統自動管理；Heap 是由你來管理！
🧠

記憶體區域解析

讓我們透過具體的程式碼範例來理解 malloc 如何在 Heap 中配置記憶體。

[圖片位置：malloc記憶體配置圖解]

程式一開始的配置

記憶體區域 描述
Stack（堆疊） 儲存區域變數 a 和指標 p（此時 p 尚未初始化）
Heap（堆積） 還沒有配置

第一次 malloc 執行時

第一次 malloc
p = (int*)malloc(sizeof(int));
  • p 取得 heap 中一塊 4 bytes 記憶體的地址（假設為 200）
  • *p = 10：將值 10 寫入 heap 的這塊記憶體
🧠 記憶體此時長這樣：

Stack:

  • a：未知值
  • p：儲存 200

Heap:

  • 200：值為 10

接下來執行 free(p);

  • heap 上 200 的空間被釋放
  • ! 注意：雖然空間釋放了，但 p 的值（仍是 200）沒被自動清除！

第二次 malloc：

第二次 malloc
p = (int*)malloc(sizeof(int));
*p = 20;
  • 系統重新配置了一塊 heap 記憶體（這次是 400）
  • 把值 20 存進 *p
🧠 記憶體現在長這樣：

Stack:

  • a：未知
  • p：儲存 400

Heap:

  • 400：值變為 20

✅ 小結

時間點 Stack 記憶體 Heap 記憶體
初始化 a 和 p 尚未賦值 無配置
第一次 malloc p → 指向 200 200 = 10
free(p) p 保留，但 200 被釋放 空間變成可再利用
第二次 malloc p 指向 400 400 = 20
