Call by Reference

記憶體區塊概念、傳址（Call by Address）與傳值（Call by Value）比較

📦 記憶體區塊

  • Heap（堆積區）
    • 用於動態記憶體配置（如 malloc、calloc）。
    • 需手動釋放；否則可能記憶體洩漏。
    • 適合大小或存活期到執行期才決定的資料。
  • Stack（堆疊區）
    • 儲存函式區域變數與參數。
    • 每次呼叫建立一個 stack frame，結束後自動回收。
  • Static / Global（靜態/全域區）
    • 全域變數與靜態變數存放區，直到程式結束才釋放。
  • Code (Text) 區
    • 儲存機器指令，通常唯讀。

🔗 傳址呼叫（Call by Address）

將變數位址傳入函式（參數型別為 int*），在函式內以 *p 改值，會直接反映在呼叫端變數上。

#include <stdio.h>

void increase(int *a){
    *a = *a + 1;
    printf("Address of a in function: %p\n", (void*)a);
}

int main(void){
    int a = 10;
    printf("Address of a in main: %p\n", (void*)&a);
    increase(&a);
    printf("%d\n", a); // a 會變成 11
    return 0;
}

📤 傳值呼叫（Call by Value）

將值的複本傳入函式（參數型別為 int），函式內對參數的更動只影響複本，不會回寫到呼叫端。

#include <stdio.h>

void increase(int a){
    a++;
    printf("Address of a in function: %p\n", (void*)&a);
}

int main(void){
    int a = 10;
    printf("Address of a in main: %p\n", (void*)&a);
    increase(a);
    printf("%d\n", a); // a 仍是 10
    return 0;
}

⚖️ 比較與常見情境

💡 選擇建議
  • 需要回傳多個結果或修改呼叫端變數 → 用 傳址（或回傳指標/結構）。
  • 僅需讀值、避免被函式更改 → 用 傳值（或 const 指標）。
  • 大型結構體建議以 const T* 傳遞，降低複製成本。

✅ 本章重點總結

  • Stack 會為每次函式呼叫建立 stack frame；位址差異可用 %p 觀察。
  • 傳址：改 *p 會改到呼叫端變數。
  • 傳值：只改到複本，不影響呼叫端。
