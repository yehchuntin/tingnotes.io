Array as Function Argument

陣列作為函式參數的機制、陣列退化、sizeof 陷阱與解決方案

❌ 錯誤寫法 - sizeof 陷阱

當陣列作為參數傳入函式時，會發生「陣列退化」(Array Decay) 現象，導致 sizeof 計算錯誤。

⚠️ 錯誤原因
  • 當陣列作為參數傳入函式時，會退化 (decay) 成指標。
  • sizeof(A) 在 sum() 裡回傳的是指標大小 (64 位元系統為 8 bytes)。
  • 實際上陣列有 5 個元素，但 size 會變成 8 / 4 = 2。
// 錯誤寫法
#include <stdio.h>

// 函式：計算陣列元素總和
int sum(int A[]) {
    int sum = 0;
    
    // ⚠️ sizeof(A) 這裡的 A 是「指標」，不是完整陣列
    // 因此 sizeof(A) 會回傳指標大小（在 64 位元系統通常是 8 bytes）
    // 而不是陣列的總大小
    int size = sizeof(A) / sizeof(A[0]); 
    
    // 印出在函式內的 A 大小
    printf("In the function, size of A: %zu bytes. Size of A[0]: %zu bytes\n", 
           sizeof(A), sizeof(A[0]));
    
    // 用迴圈累加陣列元素
    for (int i = 0; i < size; i++) {
        sum += A[i];
    }
    return sum; // 回傳總和
}

int main() {
    // 宣告並初始化陣列 A
    int A[] = {1, 2, 3, 4, 5};
    
    // 在 main 中計算陣列大小
    // sizeof(A) → 整個陣列的大小（5 × 4 bytes = 20 bytes）
    // sizeof(A[0]) → 陣列單一元素大小（int = 4 bytes）
    printf("In the main, size of A: %zu bytes. Size of A[0]: %zu bytes\n", 
           sizeof(A), sizeof(A[0]));
    
    // 呼叫 sum 函式計算總和並輸出結果
    printf("Total of A is %d\n", sum(A));
    
    return 0; 
}

📚 陣列退化現象解析

了解陣列作為函式參數時的重要特性：

🔍 重點筆記
  • 陣列在函式外：sizeof(A) 會回傳整個陣列大小 (例如 5 × 4 = 20 bytes)。
  • 陣列在函式內：A 會退化成指標，sizeof(A) 變成指標大小 (64 位元系統為 8 bytes)。
  • 傳遞陣列時：必須額外傳遞「陣列大小」進入函式。
  • 等價性：*(A + i) 和 A[i] 兩者等效。

✅ 正確版本：陣列傳遞到函式 (指標方式)

正確的做法是將陣列大小作為額外的參數傳入函式。

// 正確寫法
#include <stdio.h>

int sum(int* A, int size) { // int *A 同等於 int A[]
    int sum = 0;
    
    // 這裡的 A 是指標，因此 sizeof(A) 會回傳指標大小（64 位元系統通常為 8 bytes）
    printf("In the function, size of A: %zu bytes. Size of A[0]: %zu bytes\n",
           sizeof(A), sizeof(A[0]));
    
    for (int i = 0; i < size; i++) {
        sum += *(A + i); // *(A + i) 等同於 A[i]
    }
    
    return sum;
}

int main() {
    int A[] = {1, 2, 3, 4, 5};
    
    // 在 main() 中，A 是陣列，因此 sizeof(A) 會計算整個陣列大小（5 × 4 = 20 bytes）
    printf("In the main, size of A: %zu bytes. Size of A[0]: %zu bytes\n",
           sizeof(A), sizeof(A[0]));
    
    // 計算陣列元素個數
    int size = sizeof(A) / sizeof(A[0]); 
    
    // 將陣列首位址 &A[0] 和大小 size 傳入函式
    printf("Total of A is %d\n", sum(&A[0], size));
    
    return 0;
}

📍 Call Array by Address 詳解

當我們將陣列傳遞給函式時，實際上是傳遞陣列第一個元素的位址。

🔍 重點筆記
  • int *A 在函式參數中等同於 int A[]，陣列在傳遞時會退化為指標。
  • pow(x, 2) 回傳 double，但在這裡會被自動轉型成 int 存回陣列。
  • &A[0] 取得陣列第一個元素的位址，用來傳遞給指標參數。
  • sizeof(A) / sizeof(A[0]) 用於計算陣列長度，避免死亡陣列大小。
// call by address
#include <stdio.h>
#include <math.h>

// 函式：將陣列中每個元素平方
void twotimes(int* A, int size) {
    for (int i = 0; i < size; i++) {
        // pow(A[i], 2) 回傳 double，但會自動轉型成 int
        *(A + i) = (int)pow(*(A + i), 2);
    }
}

void show(int *p, int size) {
    for (int i = 0; i < size; i++) {
        // *(p+i) 等價於 p[i]，透過指標位移存取陣列元素
        printf("%d\t", *(p + i));
    }
    printf("\n");
}

int main() {
    int A[] = {1, 2, 3, 4, 5};
    
    // 計算陣列元素個數
    int size = sizeof(A) / sizeof(A[0]); 
    
    printf("原始陣列: ");
    show(&A[0], size);
    
    // 將每個元素平方
    twotimes(&A[0], size);
    
    printf("平方後陣列: ");
    show(&A[0], size);
    
    return 0;
}

✅ 本章重點總結

🎯 核心概念

陣列作為函式參數的核心概念是「陣列退化為指標」。這個特性讓陣列傳遞變得高效（只傳遞位址），但也要求程式設計師額外管理陣列大小資訊。

  • 陣列退化：陣列作為函式參數時會退化成指標，失去原始大小資訊。
  • sizeof 陷阱：在函式內對陣列參數使用 sizeof 只會得到指標大小。
  • 正確做法：將陣列大小作為額外參數一起傳入函式。
  • 參數等價性：int *A、int A[] 在函式參數中完全等價。
  • 傳址特性：函式內的修改會影響原陣列。
⚠️ 重要提醒

記住：在 main() 函式中 sizeof(array) 可以正確計算陣列大小，但在其他函式中的陣列參數只能得到指標大小。這是 C 語言的重要特性，務必牢記！
